Diagrama de clases

Imagen del diagrama de clases.

Este diagrama de clases representa el modelo de datos subyacente del sistema ProcessingChat. Permite entender la estructura y las relaciones entre las principales entidades del sistema.


  • Clases: Usuario, Administrador, Mensaje, Chat
  • Clase Usuario (User)
    • Atributos: user_id (entero), username (cadena de texto), email (cadena de texto), password (cadena de texto), phone (cadena de texto), rol_id (entero)
    • Relaciones: Tiene una relación de uno a uno con la clase Rol (Role)
    • Operaciones: login(), logout(), updateProfile()
  • Clase Rol (Role)
    • Atributos: role_id (entero), role_name (cadena de texto)
    • Relaciones: Tiene una relación de uno a muchos con la clase Usuario (User)
    • Operaciones: createRole(), updateRole(), deleteRole()
  • Clase Mensaje (Message)
    • Atributos: message_id (entero), chat_id (entero), user_id (entero), content (cadena de texto), timestamp (fecha y hora)
    • Relaciones: Tiene una relación de uno a uno con la clase ChatActivo (ActiveChat) y una relación de uno a uno con la clase Usuario (User)
    • Operaciones: sendMessage(), deleteMessage()
  • Clase ChatActivo (ActiveChat)
    • Atributos: chat_id (entero), chat_name (cadena de texto), client_id (entero), admin_id (entero), form_id (entero), status_id (entero), delivery_date (fecha)
    • Relaciones: Tiene una relación de uno a uno con la clase TipoFormulario (FormType) y una relación de uno a uno con la clase EstadoChat (ChatStatus). Además, tiene una relación de uno a muchos con la clase Mensaje (Message) y una relación de uno a uno con la clase Usuario (User) para el cliente y el administrador
    • Operaciones: createChat(), updateChat(), closeChat()
  • Clase TipoFormulario (FormType)
    • Atributos: form_id (entero), form_name (cadena de texto)
    • Relaciones: Tiene una relación de uno a uno con la clase ChatActivo (ActiveChat)
    • Operaciones: createFormType(), updateFormType(), deleteFormType()
  • Clase EstadoChat (ChatStatus)
    • Atributos: status_id (entero), status_name (cadena de texto)
    • Relaciones: Tiene una relación de uno a uno con la clase ChatActivo (ActiveChat)
    • Operaciones: createChatStatus(), updateChatStatus(), deleteChatStatus()
